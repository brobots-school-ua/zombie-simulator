# План: Ліміт речей які спавняться (Wishlist #22)

## Проблема
Зараз речі (ammo, bandage, medkit, матеріали з зомбі) спавняться необмежено і накопичуються на карті. Це може:
- Зменшувати FPS через багато об'єктів
- Робити гру занадто легкою (гравець завжди знайде хіл/патрони)

## Що спавниться зараз
1. **Ammo** — кожні 10-15 сек (таймер, без ліміту)
2. **Bandage** — кожні 20 сек (таймер, без ліміту)
3. **Medkit** — кожні 45 сек (таймер, без ліміту)
4. **Wood / Metal / Screws** — дропаються з зомбі (по шансу, без ліміту)

## Рішення
Додати максимальну кількість кожного типу на карті одночасно. Якщо ліміт досягнуто — новий предмет НЕ спавниться (таймер пропускає цикл).

## Ліміти
| Тип | Макс на карті |
|-----|---------------|
| Ammo | 5 |
| Bandage | 3 |
| Medkit | 2 |
| Wood | 8 |
| Metal | 8 |
| Screws | 8 |

## Як реалізувати

### Крок 1: Хелпер підрахунку в GameScene
- Додати метод `countPickupsOfType(type: PickupType): number` — рахує кількість активних пікапів цього типу в `this.pickups`

### Крок 2: Перевірка в таймерних спавнерах
- В кожному spawner callback перед створенням перевіряти: `if (this.countPickupsOfType('ammo') >= MAX) return;`
- Те саме для bandage і medkit

### Крок 3: Перевірка в дропах з зомбі
- В `onZombieKilled()` перед створенням матеріального дропу — перевірити ліміт
- Якщо вже 8 wood на карті — wood не дропається

### Зміни торкаються тільки
- `src/scenes/GameScene.ts` — додати countPickupsOfType(), перевірки в spawners і onZombieKilled()

## Порядок роботи
Одна зміна в GameScene.ts — додати метод підрахунку + перевірки в 4 місцях. Нічого більше не потрібно.
