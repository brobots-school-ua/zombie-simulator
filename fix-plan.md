# Fix Plan — Round 20: Admin Panel Fixes

## Проблеми:
1. Адмін-панель не має зручного хрестика закриття (поточний малопомітний)
2. Панель закривається коли обираєш тип зомбі в picker'і (бо `closeAdminPanel()` + `openAdminPanel()` — перерендерює всю панель)
3. Зомбі, заспавнені через адмін-панель, можуть з'явитися всередині стін

## Фікси:

### Fix 1: Покращити хрестик закриття
**Файл:** `AdminConsole.ts`
- Зробити кнопку ✕ більшою, яскравішою, з hover-ефектом
- Червоний колір для кращої видимості

### Fix 2: Панель не закривається при спавні / виборі зомбі
**Файл:** `AdminConsole.ts`
- В `openZombiePicker()`: після вибору типу зомбі — оновлювати тільки preview-блок, а НЕ перезавантажувати всю панель
- Панель залишається відкритою завжди, крім натискання ✕ або F2

### Fix 3: Адмін-спавн — зомбі не в стінах + розподіл позицій
**Файл:** `GameScene.ts` → `adminSpawnZombie()`
**Файл:** `AdminConsole.ts` → spawn handler

Логіка спавну кількох зомбі:
- 1-й зомбі — на позиції гравця
- 2-й і далі — навколо гравця з рівномірним розподілом по колу (радіус ~80px)
- Кожна позиція перевіряється через `isPositionBlocked()`
- Якщо позиція в стіні — шукати найближчу вільну по спіралі

Замість окремих викликів `adminSpawnZombie(type, x, y)` для кожного зомбі — один виклик `adminSpawnZombies(type, count)`, який сам розраховує позиції навколо гравця.

## Файли для зміни:
- `src/systems/AdminConsole.ts` — фікси 1, 2, оновити spawn handler
- `src/scenes/GameScene.ts` — фікс 3: новий метод `adminSpawnZombies(type, count)`
